The @supports rule checks whether a browser supports a specific CSS feature before applying it. This helps prevent layout issues in older browsers.

The purpose of @supports is to apply CSS styles only if the browser supports that feature.

Wrapping grid code inside @supports ensures that unsupported browsers do not break the layout.

@supports helps handle browser compatibility by providing a safe fallback layout.

This feature is useful when using modern CSS like Grid, Flexbox, or advanced properties.
